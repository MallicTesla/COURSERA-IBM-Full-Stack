Entonces, he visto en su currículum que menciona React. Por cierto, es un currículum muy impresionante. ¿Cuáles son las diferencias entre React? Y digamos JavaScript plano? Por
    ejemplo, ¿por qué nos molestaríamos con React? Bueno, JavaScript plano es un lenguaje. Por lo tanto, es similar a Python o Java. Y es más versátil, mientras que Python,
    perdón, React es una librería. Por lo tanto, con JavaScript podrías hacer mucho más. Por ejemplo, podrías hacer back end con JavaScript, React está pensado para front end.
    Y así, para el front end, las diferencias son que JavaScript es mucho más versátil. Y así, no hay como una estructura que tengas que usar, puedes estructurarlo de la manera
    que queramos. Mientras que con React, tienes que estructurar la interfaz de usuario como sea necesario. Y algo más que mencionar es, con JavaScript, todos los datos se
    almacenan en el DOM. Y así, esto hace que sea fácil perderse. Para recuperar los datos, tienes que ir, tienes que encontrar el DOM correcto. Y así, hace que sea más fácil
    perderse. Y hace que sea más difícil mantener JavaScript. Mientras que con React, simplemente se almacena en variables JavaScript normales, porque es una biblioteca de
    JavaScript. Y así, hace que sea más fácil de mantener y más fácil de usar. Aunque tienes que seguir esta cierta estructura. Lo hace más fácil en ese sentido.
